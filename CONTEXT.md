# ToramCalculator 交互模型

本上下文描述角色设计与战斗模拟共同使用的领域与交互语言。

## Language

**最小验证闭环**:
真实用户能够独立完成一次“设计、验证、分析、再设计”循环，并从前后结果中获得可解释反馈的最小产品能力；底层架构落地本身不构成闭环。
_Avoid_: 最小功能集、架构迁移闭环

**验证型玩家**:
带着明确配装、技能或战斗策略假设，使用模拟结果判断变量影响的进阶玩家；最小验证闭环首先服务此类用户。
_Avoid_: 首版新手用户、泛用户

**木桩验证场景**:
一个 Character 对一个固定练习目标的受控战斗场景；目标不移动、不攻击或施加异常，重复验证之间保持环境条件一致。
_Avoid_: 完整战斗、组队模拟

**基础能力分配**:
Character 在等级等成长条件不变时，对 STR、INT、VIT、AGI、DEX 所作的点数分配；它不包含特殊能力或由这些输入计算出的派生属性。
_Avoid_: 能力值、最终属性、派生属性

**设计差异对比**:
用户选择两条运行记录，查看它们实际使用的 Character、Simulator、操作与结果差异；系统不限制被改变的变量，也不裁定两次实验是否可比。
_Avoid_: 合法设计验证、控制变量校验、单变量因果结论

**实操验证**:
由用户发起并直接控制技能与行动完成的一次木桩验证；每次验证使用一个保持原值的设计副本重建干净世界，结果同时反映设计和本次操作表现，不能被解释为某个变量的纯因果效应。
_Avoid_: 自动流程验证、确定性配置基准测试

**运行记录对比**:
验证型玩家从多次实操验证的运行记录中选择两条进行差异分析；系统揭示输入、时长、操作和结果差异，但不替用户判定两次测试是否有效可比。
_Avoid_: 自动配对、系统裁定的有效测试

**运行结果摘要**:
一条运行记录的验证时长、总伤害和平均 DPS，以及对比时这些指标的绝对与相对差值；三项原始量必须同时呈现。
_Avoid_: 单一 DPS 排名、脱离时长的总伤害

**分析范围**:
模拟方案中定义的结果统计来源成员与目标成员集合；首个木桩验证默认统计当前主控 Player 对全部敌对成员造成的结果。
_Avoid_: 场景人数限制、当前选中目标、查询时筛选

**技能过程摘要**:
一条运行记录中的成功技能顺序，以及按技能汇总的释放次数、总伤害、伤害占比和平均单次伤害；无法归属到技能的伤害必须显式列为其他来源。
_Avoid_: 完整时间轴、自动流程一致度

**Tick 分析快照**:
逻辑引擎在每个 Tick 完成后捕获的不可变分析状态，包含属性值及其来源等时间轴依据；它独立于实时 UI 投影，只在当前 Simulator 会话内保留。
_Avoid_: UI 帧、节流快照、完整引擎检查点

**属性快照**:
Tick 分析快照中以属性路径映射属性叶节点的成员属性视图；叶节点显式包含当前值、基础值和 modifier 来源，不构造成面向展示的嵌套树。
_Avoid_: 嵌套属性树、未定义结构的属性集合

**模拟方案**:
Simulator 持久化并跨端同步的完整设计，包括初始场景、成员配置与流程、随机种子、手动控制配置和分析范围；它完整定义用户想要运行什么，不保存某次运行产生的结果。
_Avoid_: 运行记录、页面状态、运行时附加参数

**设计副本**:
当前 Simulator 会话中一份完整、可运行的模拟方案副本；它产生运行记录后保持原值，后续编辑形成新副本，用户可以选择其中一个把差异应用到正式模拟方案。
_Avoid_: Character 版本、运行记录、普通数据缓存

**成员流程**:
Simulator 成员在当前模拟方案中可选的自动行动设计；同一个 Character 在不同模拟方案中可以拥有不同成员流程，未设置流程的 Player 由用户手动控制。
_Avoid_: Mob 固有 AI、Character 行为、运行输入记录

**输入轨迹**:
一次实操验证中用户发出的语义输入及其接纳结果；它包含被拒绝的输入，只属于本次运行分析，不直接构成成员流程。
_Avoid_: 成员流程、行动录制

**行动录制**:
一次实操验证中被成员运行时真正接纳的语义行动及其相对运行起点 Tick；用户可以显式把它转换为成员流程，被拒绝的输入不属于行动录制。
_Avoid_: 原始点击记录、运行结果、自动保存的成员流程

**Mob 固有 AI**:
Mob 模板自身定义的自主战斗规则；所有引用该模板的 Mob 成员共享这套规则，Simulator 成员不复制或覆盖它。
_Avoid_: 成员流程、木桩临时行为

**交互空间**:
一组具有一致行为规则和共享交互语义的用户活动范围；它不等同于页面或路由，一个路由可以包含多个交互空间。
_Avoid_: 页面状态、路由阶段

**AUI 行为状态**:
不同输入与表现模态共同解释的当前交互模式及其动态目标；整个应用在任一时刻只有一份有效事实。
_Avoid_: 视觉意图、注意力状态、界面形态

**引擎运行时宿主**:
主线程内拥有长期计算 Worker 与分支执行资源的应用级生命周期边界；它不拥有设计副本、业务阶段或界面投影状态。
_Avoid_: 主线程引擎服务、引擎会话

**Simulator 会话**:
围绕一个模拟方案持续进行设计、验证与分析的交互式工作上下文；同一应用最多只有一个活跃会话，只有显式结束或切换方案才会终止它。
_Avoid_: Simulator 页面、全局应用会话

**Simulator 会话生命周期**:
Simulator 会话从不存在、装载、活跃到结束的存在性变化；它不包含业务阶段或模拟执行状态。
_Avoid_: 引擎状态、业务阶段

**会话读面**:
Simulator 会话向 UI、HUD 与场景投影公开的当前只读事实集合；它不包含设计副本本体或逐帧世界数据。
_Avoid_: 会话快照、页面状态

**模拟任务**:
为 Wiki 或 Character 技能预览执行的一次有界、无交互模拟计算；多个任务可以并行，但都不拥有业务阶段或持续交互进度。
_Avoid_: Simulator 会话、后台会话

**实时视图**:
模拟运行时面向 UI、HUD 与场景的最新世界投影；允许节流、合并或丢弃已经过时的中间帧。
_Avoid_: 运行记录、逐帧历史

**运行记录**:
当前 Simulator 会话中一次实操验证的分析依据；它关联产生自己的设计副本，并包含输入轨迹、行动录制、结果摘要和 Tick 分析快照，会随会话刷新或结束而丢失。
_Avoid_: 实时视图、页面帧缓存

**业务阶段**:
当前 Simulator 会话在“设计、验证、分析”循环中的位置；它不属于整个用户或整个应用。
_Avoid_: 全局阶段、页面、路由、当前界面

**模拟执行状态**:
Worker 中当前模拟运行时所处的未初始化、运行、暂停、停止或失败状态；它独立于业务阶段，只能作为执行事实被投影。
_Avoid_: 业务阶段、Simulator 会话生命周期

**Wiki**:
独立于 Simulator 会话业务阶段的知识参考面；打开或关闭 Wiki 不改变或暂停 Simulator 会话。

**Character 工作面**:
用于配置与预览角色的独立工作面；打开它会切换当前界面与场景投影，但不改变、暂停或结束 Simulator 会话。
_Avoid_: Character 阶段、Simulator 子阶段

**Simulator 成员**:
某个 Simulator 独占的一项参战配置；Player 类型成员明确选择一个 Character，Mob 类型成员明确选择 Mob 模板及该成员的难度配置，支持方案内自动行动的成员还可以拥有成员流程。
_Avoid_: 共享成员、Player 当前 Character

**Character 引用**:
Player 类型 Simulator 成员对共享 Character 的明确关系；设计副本包含该 Character 当时的完整设计值，应用副本差异时更新原 Character。
_Avoid_: Player 当前 Character、隐式克隆 Character

**Mob 引用**:
Mob 类型 Simulator 成员对 Mob 模板的明确关系及该成员的难度配置；设计副本包含运行所需的模板值和资源描述。
_Avoid_: Mob 模板副本、运行中查询模板

**场景解析**:
把设计副本一次派生为同一设计身份的逻辑世界输入与静态世界资源；它不创建另一份可编辑或长期保存的设计事实。
_Avoid_: 引擎加载、渲染命令转换

**世界资源**:
3D 场景创建实体所需的静态视觉描述，包括成员对应的模型、外观和动画资源；它不包含位置、动作或生命状态等模拟动态。
_Avoid_: 设计副本、动态渲染命令

**重复成员配置**:
不同 Simulator 各自拥有内容相同的成员配置，或各自引用同一个 Character/Mob 资源；这些成员身份彼此独立，不会因修改一方编排而改变另一方。
_Avoid_: 共享 Member、共享 Team
