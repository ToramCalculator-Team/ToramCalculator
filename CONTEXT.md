# ToramCalculator 交互模型

本上下文描述角色设计与战斗模拟共同使用的领域与交互语言。

## Language

**交互空间**:
一组具有一致行为规则和共享交互语义的用户活动范围；它不等同于页面或路由，一个路由可以包含多个交互空间。
_Avoid_: 页面状态、路由阶段

**AUI 行为状态**:
不同输入与表现模态共同解释的当前交互模式及其动态目标；整个应用在任一时刻只有一份有效事实。
_Avoid_: 视觉意图、注意力状态、界面形态

**引擎运行时宿主**:
主线程内拥有长期计算 Worker 与分支执行资源的应用级生命周期边界；它不拥有方案快照、业务阶段或界面投影状态。
_Avoid_: 主线程引擎服务、引擎会话

**Simulator 会话**:
围绕一个模拟方案持续进行设计、验证与分析的交互式工作上下文；同一应用最多只有一个活跃会话，只有显式结束或切换方案才会终止它。
_Avoid_: Simulator 页面、全局应用会话

**Simulator 会话生命周期**:
Simulator 会话从不存在、装载、活跃到结束的存在性变化；它不包含业务阶段或模拟执行状态。
_Avoid_: 引擎状态、业务阶段

**会话读面**:
Simulator 会话向 UI、HUD 与场景投影公开的当前只读事实集合；它不包含运行快照本体或逐帧世界数据。
_Avoid_: 会话快照、页面状态

**模拟任务**:
为 Wiki 或 Character 技能预览执行的一次有界、无交互模拟计算；多个任务可以并行，但都不拥有业务阶段或持续交互进度。
_Avoid_: Simulator 会话、后台会话

**设计草稿**:
Simulator 会话在设计阶段维护的可变模拟输入；它表达用户准备在下一次验证中使用的设计。
_Avoid_: 运行快照、持久模拟方案

**运行快照**:
Simulator 会话从设计进入验证时捕获的不可变模拟输入；本次验证与分析始终使用它，不受后续设计或 Character 修改影响。
_Avoid_: 设计草稿、最新方案、实时角色数据

**实时视图**:
模拟运行时面向 UI、HUD 与场景的最新世界投影；允许节流、合并或丢弃已经过时的中间帧。
_Avoid_: 运行记录、逐帧历史

**运行记录**:
一次验证运行产生的有界、有序且不可静默丢失的分析依据；它独立于实时视图的消费速度。
_Avoid_: 实时视图、页面帧缓存

**业务阶段**:
当前 Simulator 会话在“设计、验证、分析”循环中的位置；它不属于整个用户或整个应用。
_Avoid_: 全局阶段、页面、路由、当前界面

**模拟执行状态**:
Worker 中当前模拟运行时所处的未初始化、运行、暂停、停止或失败状态；它独立于业务阶段，只能作为执行事实被投影。
_Avoid_: 业务阶段、Simulator 会话生命周期

**Wiki**:
独立于 Simulator 会话业务阶段的知识参考面；打开或关闭 Wiki 不改变或暂停 Simulator 会话。

**Character 工作面**:
用于配置与预览角色的独立工作面；打开它会切换当前界面与场景投影，但不改变、暂停或结束 Simulator 会话。
_Avoid_: Character 阶段、Simulator 子阶段

**Simulator 成员**:
某个 Simulator 独占的一项参战配置；Player 类型成员明确选择一个 Character，Mob 类型成员明确选择 Mob 模板及该成员的难度配置。
_Avoid_: 共享成员、Player 当前 Character

**Character 引用**:
Player 类型 Simulator 成员在设计草稿中保存的 `characterId`；进入验证时才解析该 Character 的最新持久数据并冻结进运行快照。
_Avoid_: Character 副本、运行中角色数据

**重复成员配置**:
不同 Simulator 各自拥有内容相同的成员配置，或各自引用同一个 Character/Mob 资源；这些成员身份彼此独立，不会因修改一方编排而改变另一方。
_Avoid_: 共享 Member、共享 Team
