# ToramCalculator 交互模型

本上下文描述机体设计与战斗模拟共同使用的领域与交互语言。

## Language

**游戏档案（GameProfile）**:
一个用户在某个游戏领域内拥有的全部资产集合，包括该游戏的账号数据、角色数据以及参与维护的 Wiki 数据；它不等同于登录身份或单个游戏账号。
_Avoid_: OAuth Account、PlayerProfile、单个存档

**游戏账号（GameAccount）**:
游戏档案中的一份具体游戏账号数据集合；它组织该账号拥有的角色及其他账号私有资产，具体结构由游戏领域定义。
_Avoid_: 登录账号、GameProfile、Character

**托拉姆账号（ToramAccount）**:
托拉姆领域对游戏账号的具体定义，拥有该账号下的机体、自定义装备、宠物与外观等数据。
_Avoid_: Player、平台账号、GameProfile

**机体（Character）**:
托拉姆账号中可独立识别、配置并参与模拟的角色实体；代码中的 `Character` 指向这个概念。
_Avoid_: 托拉姆角色、Player、Simulator Member

**机体配置**:
一个机体可保存的业务数据，包括成长属性、装备、技能与其他角色配置。
_Avoid_: Character 数据、控件状态、Simulator Member 配置

**当前机体配置**:
机体编辑器当前用于显示和验证的已保存机体配置；合法编辑只有在自动保存完成后才成为当前配置，后续同步纠正会替换它。
_Avoid_: 控件显示值、待执行编辑、服务端确认值

**最小验证闭环**:
真实用户能够独立完成一次“设计、验证、分析、再设计”循环，并从前后结果中获得可解释反馈的最小产品能力；底层架构落地本身不构成闭环。
_Avoid_: 最小功能集、架构迁移闭环

**验证型玩家**:
带着明确配装、技能或战斗策略假设，使用模拟结果判断变量影响的进阶玩家；最小验证闭环首先服务此类用户。
_Avoid_: 首版新手用户、泛用户

**木桩验证场景**:
一个机体对一个固定练习目标的受控战斗场景；目标不移动、不攻击或施加异常，重复验证之间保持环境条件一致。
_Avoid_: 完整战斗、组队模拟

**基础能力分配**:
机体在等级等成长条件不变时，对 STR、INT、VIT、AGI、DEX 所作的点数分配；它不包含特殊能力或由这些输入计算出的派生属性。
_Avoid_: 能力值、最终属性、派生属性

**设计差异对比**:
用户选择两条运行记录，查看它们实际使用的机体、Simulator、操作与结果差异；系统不限制被改变的变量，也不裁定两次实验是否可比。
_Avoid_: 合法设计验证、控制变量校验、单变量因果结论

**实操验证**:
由用户发起并直接控制技能与行动完成的一次木桩验证；每次验证使用一个保持原值的设计副本重建干净世界，结果同时反映设计和本次操作表现，不能被解释为某个变量的纯因果效应。
_Avoid_: 自动流程验证、确定性配置基准测试

**运行记录对比**:
验证型玩家从多次实操验证的运行记录中选择两条进行差异分析；系统揭示输入、时长、操作和结果差异，但不替用户判定两次测试是否有效可比。
_Avoid_: 自动配对、系统裁定的有效测试

**运行结果摘要**:
Simulator 业务分析模块根据一条运行记录关联的设计副本与引擎运行产出派生出的验证时长、总伤害和平均 DPS，以及对比时这些指标的绝对与相对差值；三项原始量必须同时呈现。
_Avoid_: 单一 DPS 排名、脱离时长的总伤害

**分析范围**:
模拟方案中定义的结果统计来源成员与目标成员集合；首个木桩验证默认统计当前主控 Player 对全部敌对成员造成的结果。
_Avoid_: 场景人数限制、当前选中目标、查询时筛选

**技能过程摘要**:
一条运行记录中的成功技能顺序，以及按技能汇总的释放次数、总伤害、伤害占比和平均单次伤害；无法归属到技能的伤害必须显式列为其他来源。
_Avoid_: 完整时间轴、自动流程一致度

**Tick 分析快照**:
逻辑引擎在每个 Tick 完成后的不可变分析状态，包含属性值及其来源等时间轴依据；它是必须能被精确还原的查询语义，不要求每个 Tick 长期物化为完整对象，并且独立于实时 UI 投影，只在当前 Simulator 会话内保留。
_Avoid_: UI 帧、节流快照、完整引擎检查点

**属性快照**:
Tick 分析快照中以属性路径映射属性叶节点的成员属性视图；叶节点显式包含当前值、基础值和 modifier 来源，不构造成面向展示的嵌套树。
_Avoid_: 嵌套属性树、未定义结构的属性集合

**模拟设计（SimulationDesign）**:
Simulator 产品中持久化并跨端同步的一份完整设计，包括初始场景、成员配置与流程、随机种子、手动控制配置和分析范围；它完整定义用户想要运行什么，不保存某次运行产生的结果。
_Avoid_: Simulator 实体、运行记录、页面状态、运行时附加参数

**设计副本（DesignCopy）**:
当前 SimulatorSession 中一份完整、可运行的 SimulationDesign 副本；它产生运行记录后保持原值，后续编辑形成新副本，用户可以选择其中一个把差异应用到正式 SimulationDesign。
_Avoid_: 机体版本、运行记录、普通数据缓存

**成员流程**:
Simulator 成员在当前模拟方案中可选的自动行动设计；同一个机体在不同模拟方案中可以拥有不同成员流程，未设置流程的 Player 由用户手动控制。
_Avoid_: Mob 固有 AI、机体行为、运行输入记录

**输入轨迹**:
一次实操验证中用户发出的语义输入及其接纳结果；它包含被拒绝的输入，只属于本次运行分析，不直接构成成员流程。
_Avoid_: 成员流程、行动录制

**行动录制**:
一次实操验证中被成员运行时真正接纳的语义行动及其相对运行起点 Tick；用户可以显式把它转换为成员流程，被拒绝的输入不属于行动录制。转换必须从产生该录制的运行记录所关联的设计副本创建新副本，不能隐式写入当前选中的其他设计副本。
_Avoid_: 原始点击记录、运行结果、自动保存的成员流程

**Mob 固有 AI**:
Mob 模板自身定义的自主战斗规则；所有引用该模板的 Mob 成员共享这套规则，Simulator 成员不复制或覆盖它。
_Avoid_: 成员流程、木桩临时行为

**交互空间**:
一组具有一致行为规则和共享交互语义的用户活动范围；它不等同于页面或路由，一个路由可以包含多个交互空间。
_Avoid_: 页面状态、路由阶段

**AUI 行为状态**:
不同输入与表现模态共同解释的当前交互模式及其动态目标；整个应用在任一时刻只有一份有效事实。它可以约束多个交互空间，但不包含领域数据和具体视觉表现状态。
_Avoid_: 视觉意图、注意力状态、界面形态

**Simulator 会话**:
围绕一个已加载模拟方案持续管理设计、验证与分析的交互会话；同一应用最多只有一个已加载会话。离开 Simulator 页面不会结束它，只有显式结束或切换方案才会清除其内容；当路由目标与当前会话不一致时，用户必须显式选择返回当前会话或切换目标方案，路由不能自行替换会话。
_Avoid_: Simulator 页面、全局应用会话

**机体会话（CharacterSession）**:
围绕一个正式机体持续管理身份、切换、编辑与验证的交互会话；同一时刻最多装载一个机体。离开机体编辑器不会自动结束它，它也不读取或修改 Simulator 设计副本。
_Avoid_: 机体编辑器页面状态、机体编辑器、Simulator 机体会话

**机体配置自动保存**:
用户形成合法机体配置修改后，系统自动保存并明确反馈失败；不存在机体级保存或取消操作，也不形成长期未提交的领域草稿。
_Avoid_: 手动保存机体配置、机体配置草稿应用、静默写入失败

**运行记录**:
当前 Simulator 会话中一次实操验证的完整业务记录；它关联产生自己的设计副本，并组织输入轨迹、行动录制和结果分析。它持有按 Tick 还原分析状态的唯一历史数据源，查询结果不形成第二份长期历史；记录会随会话刷新或结束而丢失。
_Avoid_: 实时视图、页面帧缓存、持久运行历史

**Simulator 业务阶段**:
应用围绕当前 Simulator 会话在“设计、验证、分析”循环中的稳定位置；它可以约束 Simulator 之外的交互空间，但打开机体编辑器或 Wiki 不会自动改变该阶段。开始或结束验证命令尚未完成不构成额外阶段。
_Avoid_: Session 私有阶段、页面、路由、当前界面

**Wiki**:
独立于 Simulator 会话业务阶段的知识参考面；打开或关闭 Wiki 不改变或暂停 Simulator 会话。

**机体编辑器**:
AUI 中用于维护正式机体配置并对当前机体配置执行属性、技能等验证的独立交互空间；它不读取或修改 Simulator 设计副本，打开它也不会改变、暂停或结束 Simulator 会话。
_Avoid_: 机体阶段、Simulator 子阶段、DesignCopy 机体编辑器、Character 工作面

**Simulator 成员**:
某个 Simulator 独占的一项参战配置；Player 类型成员明确选择一个机体，Mob 类型成员明确选择 Mob 模板及该成员的难度配置，支持方案内自动行动的成员还可以拥有成员流程。
_Avoid_: 共享成员、Player 当前机体

**机体引用**:
Player 类型 Simulator 成员对共享机体的明确关系；设计副本包含该机体当时的完整配置值，应用副本差异时更新原机体。
_Avoid_: Player 当前机体、隐式克隆机体

**Mob 引用**:
Mob 类型 Simulator 成员对 Mob 模板的明确关系及该成员的难度配置；设计副本包含运行所需的模板值和资源描述。
_Avoid_: Mob 模板副本、运行中查询模板
