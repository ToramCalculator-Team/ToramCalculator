# 技能类型分析（2026-05-22）

## 结论

`skill_variant` 的普通录入入口采用“释放方式 + 行为类型 + 类型参数”。

行为树保留为运行时 IR 和高级自定义入口，普通用户上传技能时优先选择默认行为模板。

默认行为模板收敛为 6 类：

| 模板族 | 目的 | 默认行为树覆盖范围 |
|---|---|---|
| `DamageAction` | 主动伤害流程 | 单段、多段、`DAMAGE_RANGE_TYPE` 范围、持续区域命中、命中异常、简单动作后状态/回复 |
| `StatusAction` | 主动状态流程 | 自身 buff、队伍 buff、目标 debuff、光环 |
| `RecoveryAction` | 主动回复流程 | HP/MP 即时回复、目标/队伍回复、简单周期回复 |
| `PassiveRule` | 战前安装规则 | 常驻属性、现有 RuntimeAttachment 形态的 pipeline overlay、订阅、阈值 watcher、技能分支激活器 |
| `WorldObject` | 独立世界对象 | 分身、召唤物、陷阱对象 |
| `WorldZone` | 独立世界区域 | 结界、领域、陷阱区域、持续区域 |

`CustomBehaviorTree` 作为高级模式覆盖顺序、条件、循环、动态插入动作、低频惩罚、复杂层数和复杂事件处理。

资料型、生产型、纯说明技能进入展示/导入层，不进入战斗运行时行为分类。

类型结构见 [skillBehaviorTypes.ts](./skillBehaviorTypes.ts)。

## 释放方式与行为类型

`skillType` 描述释放入口：

| skillType | 默认效果数量 | 含义 |
|---|---:|---|
| `瞬發` | 280 | 立即进入动作流程 |
| `被動` | 160 | 战前安装或运行时被动订阅 |
| `須詠唱` | 39 | 释放前有咏唱阶段 |
| `須蓄力` | 25 | 释放前有蓄力阶段 |
| `EX技能` | 10 | 生产、系统或特殊技能 |

`DamageAction / StatusAction / RecoveryAction / PassiveRule / WorldObject / WorldZone / CustomBehaviorTree` 描述运行时行为。

推荐 `skill_variant` 保存三组字段：

| 字段 | 类型 | 含义 |
|---|---|---|
| `executionKind` | `Active / Passive / Ex` | 技能入口类型 |
| `chargingType` | `SkillChargingType` | 咏唱、蓄力或无读条 |
| `behaviorKind` | `SkillBehaviorKind` | 默认行为模板类型 |
| `behaviorParams` | `SkillBehaviorParams` | 模板参数 |
| `customBehaviorTreeId` | `string?` | 高级自定义行为树 |

## 默认参数准入阈值

默认模板参数只收纳同构运行时流程在 514 条默认效果中出现次数 `> 5` 的模式。

出现次数 `<= 5` 的具体流程保存为展示说明，并通过 `CustomBehaviorTree` 实现运行时。

统计粒度按具体运行时流程计算。例如“HP 不足时剩 1HP 并降级”和“HP 不足时昏厥”分别统计。

`stack`、`next`、`damage_stat` 这类分支名只能作为导入线索；是否进入默认模板取决于流程是否同构。

## 资源职责边界

基础 HP/MP 技能消耗归 `skill.cost` 管线与玩家状态机。

默认主动行为模板从“技能已被接受且基础消耗已支付”的上下文开始。

`skill.cost` 表达技能释放门槛；HP/MP 无法支付时，玩家状态机拒绝发动技能。

效果阶段的 HP/MP 改变归具体行为模板或自定义行为树。

破风之势、献祭这类技能记录为 `hpCost=0`；效果阶段扣 HP 与不足惩罚按低频特殊流程进入 `CustomBehaviorTree`。

魔法火神这类连续扣 MP 流程按持续行为处理；实测中技能 MP 消耗修正不会影响后续持续扣除，所以该扣除流程不进入 `skill.cost`。

## 伤害事件语义

`DamageAction` 默认行为树：

```text
DamageAction {
  ActiveLifecycle.executeTemplateBody {
    for each damageEvent in damageEvents {
      wait(damageEvent.timing.startDelayMs)
      snapshotCasterForDamagePayload
      inferDamageTagsFrom(damageApplication, damageResolution, formula.element, ailments)
      buildDamageAreaRequest(damageEvent.formula, damageEvent.delivery, damageEvent.timing)
      registerDamageAreaRequest
    }
    runFollowUpStatusesAndRecoveries
  }
}
```

伤害模板只创建 `DamageAreaRequest`。

`DamageAreaSystem.tick` 负责范围查询、再次命中节流、距离/目标数注入，并向目标成员派发 `受到攻击`。

目标侧 FSM 接收 `DamageDispatchPayload` 后按 `hitCheck → damageCalc → applyDamage` 处理命中、伤害和 HP/MP 写回。

单次技能内的多段伤害保存为多个 `DamageEventParam`；每个事件单独进行一次命中判定。

`DamageHitPolicyParam` 已删除；`judgment / hitCount / damageCount` 不进入 skill_variant 表单。

`DamageTimingParam` 只保留：

| 字段 | 含义 |
|---|---|
| `startDelayMs` | 登记伤害区域前等待的时间 |
| `durationMs` | 伤害区域存续时间 |

周期次数、结束条件、循环插入和条件续发进入 `CustomBehaviorTree`。

`damageTags` 由模板推导，来源包括伤害施加类型、结算类型、元素公式结果、异常衍生伤害和范围警告区。

`element` 使用公式表达；示例包括固定元素、主手元素、副手元素、元素所及影响后的元素。

`DamageStatOverrideParam` 已删除；常见公式差异写入 `damageFormula`，改写 `damageCalc` 内部变量的技能使用现有 pipeline overlay 或自定义行为树。

## 命中异常

`AilmentParam` 只保留：

| 字段 | 含义 |
|---|---|
| `type` | `AbnormalType` 枚举值 |
| `chance` | 命中后判定概率公式 |

异常判定时机统一为命中时。

资料站异常名称在导入阶段映射到 `AbnormalType`；无法映射时进入导入错误或展示说明。

## 状态模板

`StatusEffectParam` 已删除。

`StatusAction` 直接消费 `StatusBehaviorParams`：

```text
StatusAction {
  ActiveLifecycle.executeTemplateBody {
    recipients = resolveRecipients(scope)
    for each recipient in recipients {
      for each modifier in modifiers {
        value = evaluate(modifier.value)
        recipient.statContainer.addModifier(modifier.attr, modifier.targetType, value, source)
      }
      if durationMs exists then scheduleRemoveModifiersBySource
      if aura exists then registerAura
    }
  }
}
```

`StatModifierParam` 对齐 `StatContainer.addModifier(attr, targetType, value, source)`。

`source` 缺省时由模板按技能、buff、debuff 或 passive 来源生成。

状态结束通过来源 ID 清理 modifier。

## 回复模板

`RecoveryParam` 使用单个 `amount` 公式。

`constant / extraValue / extraText` 已删除；固定回复、额外回复、条件加成都写入同一个公式。

周期回复使用互斥调度结构：

| 调度 | 字段 |
|---|---|
| `Immediate` | 无额外字段 |
| `Periodic` | `durationMs`、`intervalMs` |

周期次数和终止条件进入 `CustomBehaviorTree`。

## 被动与触发

`PassiveRule` 复用现有 `RuntimeAttachment` 与托环数据形态：

| 字段 | 对应运行时 |
|---|---|
| `attributeSlots` | `StatContainer` 构造前的属性槽合并 |
| `modifiers` | `StatContainer.addModifier` |
| `pipelinePatches` | `PipelineOverlay` |
| `subscriptions` | `ProcBus` |
| `thresholdWatchers` | `AttributeWatcherRegistry` |
| `skillBranchActivators` | `CommonCondition.isSkillBranchActivated` |

`TriggerParam` 已删除；事件触发参考雷吉斯托环的 `EventSubscriptionEffect`。

`PassiveModifierParam` 已删除；被动与状态共用 `StatModifierParam`。

`nextSkillModifiers` 已删除；下一技能修正若能表达为 `skill.cost` 或其他公开 pipeline overlay，则走 `pipelinePatches`，流程插入型下一技能效果进入 `CustomBehaviorTree`。

## 状态槽与计时器

新增运行时数据通过 `AttributeSlotDeclarationData.path` 声明到 `StatContainer`。

路径就是槽身份，例如：

```text
skill.crossFire.stack
passive.hpEmergency.lastTriggeredFrame
buff.blessing.startedAtMs
```

默认模板只保留 `StateTimerParam` 这种计时器形态的跨帧状态。

层数增减、分支选择、消耗层数后执行不同动作、状态转移图进入 `CustomBehaviorTree`。

## 世界对象与世界区域

`WorldObjectOrZoneParams` 已拆分为两类：

| 类型 | 语义 | 关键字段 |
|---|---|---|
| `WorldObjectBehaviorParams` | 独立实体或对象 | `objectKind`、`spawnAt`、`durationMs`、`objectBehaviorTreeId` |
| `WorldZoneBehaviorParams` | 空间区域 | `zoneKind`、`center`、`radius`、`durationMs`、`tickIntervalMs`、`zoneBehaviorTreeId` |

对象与区域的生命周期、空间索引、checkpoint 会触碰 `World/Area`、`MemberManager` 和线程消息，首批实现优先级低于伤害、状态、回复、被动。

| 技能 | 模板族 | 参数匹配 |

## 表单策略

上传技能时先选择 `executionKind`，再选择 `chargingType` 和 `behaviorKind`。

表单按行为类型渲染：

| 行为类型 | 表单字段 |
|---|---|
| `DamageAction` | 伤害事件、常数、倍率、元素公式、伤害类型、惯性、`DamageRangeType`、范围参数、异常、简单后续状态/回复 |
| `StatusAction` | 作用域、持续时间、modifier、光环 |
| `RecoveryAction` | HP/MP、目标、回复公式、立即/周期调度 |
| `PassiveRule` | 属性槽、modifier、RuntimeAttachment 高级能力 |
| `WorldObject` | 对象类型、位置、持续时间、对象行为树 |
| `WorldZone` | 区域类型、中心、半径、持续时间、周期、区域行为树 |
| `CustomBehaviorTree` | 行为树编辑器与暴露参数 |

普通表单隐藏 `pipelinePatches / subscriptions / thresholdWatchers / skillBranchActivators` 的原始 JSON 编辑入口；维护者或高级模式使用这些字段。

## 迁移顺序

1. 增加 `executionKind / chargingType / behaviorKind / behaviorParams / customBehaviorTreeId`，保留旧行为字段。
2. 为 `DamageAction / StatusAction / RecoveryAction / PassiveRule` 写内置模板生成器。
3. 保存新表单时生成兼容旧运行时的行为树或旧字段。
4. 补 `WorldObject / WorldZone` 运行时服务后接入对应模板。
5. 增加 `resolveSkillBehavior(skillVariant)`，运行时优先读取新字段，缺失时回退旧行为树字段。
6. 数据迁移稳定后再清理旧字段。

## 风险

`behaviorParams` 一旦允许任意步骤列表，就会退化成低配行为树。

默认类型只暴露领域事实：公式、范围、持续时间、modifier、订阅、阈值 watcher、对象/区域几何。

当录入内容需要顺序、条件、循环、状态转移、动态插入动作时，使用 `CustomBehaviorTree`。
