# 0024 - 交互式 Simulator 会话与模拟任务分离

- **状态**: Accepted
- **日期**: 2026-07-10
- **决策层**: 跨层（应用状态 / 引擎 / UI）
- **相关代码**: `src/machines/`、`src/routes/(app)/(features)/simulator/`、`src/lib/engine/core/thread/`
- **相关 ADR**: 补充 0015、0021、0025

## 背景

主 Simulator、Wiki 计算和 Character 技能预览都会运行战斗模拟，但它们的生命周期与交互语义不同。主 Simulator 需要保留用户的阶段、控制关系和运行进度；Wiki 与技能预览需要同时发起多个有限计算。如果把二者都称为会话，要么页面切换会销毁主业务进度，要么“一次只能一个会话”会错误限制并行计算。

## 候选方案

### A. 所有模拟统一为页面级会话

- 优点：页面挂载时创建、卸载时销毁，资源回收直接。
- 缺点：打开 Wiki 或 Character 会丢失主 Simulator 进度；并行预览也会被迫承担业务阶段和长期运行态。

### B. 保留一个交互式会话，有限计算建模为任务

- 优点：主业务连续性与并行计算相互独立；两类运行各自只有一套清晰生命周期。
- 缺点：会话所有权必须迁出页面，应用还需要显式管理会话终止和任务资源上限。

## 决议

选 B：**同一应用最多保留一个交互式 Simulator 会话；Wiki 与 Character 技能预览等并行计算是模拟任务，不是 Simulator 会话。**

1. Simulator 会话围绕一个已加载模拟方案持续存在，拥有该方案身份、设计/验证/分析业务阶段、交互控制关系和运行进度。
2. Simulator 会话不由页面生命周期拥有。打开或关闭 Wiki、进入或离开 Character 工作面时，会话完整保留；只有显式结束会话或切换模拟方案才会替换它。
3. 应用生命周期内只有一台 AUI 行为状态机保存跨模态共享的交互事实和当前 Simulator 会话的阶段事实；引擎运行态与领域数据仍由各自模块拥有。
4. Simulator 的 UI、HUD 与场景内容是可独立卸载和恢复的投影；投影卸载不得重置引擎、解绑控制器或清除阶段。
5. 模拟任务是有界、无交互、输入输出明确的计算。它不拥有业务阶段、主控关系、实时世界或可恢复的交互进度，完成、失败或取消后即可释放。
6. 模拟任务可以在没有 Simulator 会话时运行，也可以受资源上限约束地与当前会话及其他任务并行运行。它们不得读写当前 Simulator 会话的可变运行态。

## 代价

主 Simulator 会话会在界面不可见时继续占用运行时资源，必须提供明确的结束和替换语义。模拟任务与会话走不同生命周期，调用方必须选择正确能力；后台任务还需要统一的并发、取消和过期结果丢弃策略。

## 影响范围

- Simulator 页面从会话所有者降为输入与投影边界，页面卸载不再等价于会话结束。
- Character 技能预览继续通过有限模拟任务并行计算；Wiki 模拟复用同一任务能力，不创建额外 Simulator 会话。
- “最多一个”只约束交互式 Simulator 会话，不约束受 Worker 池容量控制的模拟任务数量。
- 会话的设计草稿与运行快照边界遵守 ADR 0025。
- 迁移与验收见 `docs/plans/0021-aui-interface-state-migration.md`。
