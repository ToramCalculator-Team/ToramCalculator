# 0043 - 行动录制从最终输入记录派生

- **状态**: Accepted
- **日期**: 2026-07-15
- **决策层**: 跨层（输入 / 引擎运行产出 / 分析 / 成员流程）
- **相关 ADR**: Supersedes 0033

## 决策问题

ADR 0033 要求分别保存输入轨迹和行动录制，但两者会重复维护同一次行动接纳。系统需要决定继续保存两组事实，还是只保存具有最终判决的输入记录，并从被接纳记录派生行动录制。

## 决策驱动

- 每个输入只能有一个权威最终结果。
- 分析必须保留被拒绝输入，成员流程只能消费被接纳输入。
- 运行分析和成员流程生成必须读取同一份不可变事实。
- 缺失关键事实不能通过默认值或静默过滤掩盖。

## 候选方案

### A. 分别保存输入轨迹和行动录制

- 优点：行动录制可以直接读取并拥有独立结构。
- 缺点：同一次接纳需要双写，两组身份、时间和行动事实可能漂移。

### B. 保存最终输入记录并派生行动录制

- 优点：分析和流程生成共享同一事实，不需要双写一致性协议。
- 缺点：最终输入契约必须包含忠实派生所需的全部事实。

## 决议

选择 B：**运行产出只保存一组具有最终判决的输入记录；行动录制是其中被接纳输入的派生视图，不物理保存第二份事实。**

- 被接纳记录必须包含忠实派生行动及其运行时序所需的已解析事实，不能在运行结束后从当前状态反推。
- 被拒绝输入继续用于运行分析，但不进入行动录制或成员流程。
- 未形成最终判决或缺少派生所需事实时，运行产出边界必须显式失败，不能补默认值或静默丢弃。
- 分析和成员流程生成共享同一派生边界，不分别维护同形筛选或缓存。
- 行动录制仍是会话内运行产出的视图，只有用户显式保存时才能写入持久成员流程。

## 代价

最终输入契约需要长期携带多个消费者共同需要的最小事实，协议变化必须同时验证分析和成员流程生成。若行动录制以后拥有独立编辑、注释或版本语义，它将不能继续只作为输入记录的派生视图。

## 重新评估条件

- 行动录制需要保存无法从最终输入事实表达的独立用户语义。
- 保存前编辑的行动录制需要独立身份和生命周期。
- 最终输入契约无法以可接受成本保留忠实派生所需的事实。
