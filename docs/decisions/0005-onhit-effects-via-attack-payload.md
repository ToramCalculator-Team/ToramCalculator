# 0005 - 命中后效果通过攻击 Payload 传递

- **状态**: Proposed
- **日期**: 2026-05-29
- **决策层**: 编排层 / 通信
- **相关代码**: `src/lib/engine/core/World/Member/runtime/Agent/CommonActions.ts`、`src/lib/engine/core/World/Member/runtime/DamageResolution.ts`
- **相关 ADR**: 0004

## 背景

大量技能具有“命中后施加 buff/异常”的固定模式。若每个技能都用行为树显式注册 proc、发起攻击并等待异步回调，树结构会变得冗长，也不利于从资料自动生成。

## 候选方案

### A. 行为树显式编排 subscribeProc + attack

- 优点：复用既有机制，能表达复杂命中分支。
- 缺点：重复、难生成，命中后效果脱离攻击声明。

### B. 攻击 Payload 携带 onHitEffects

- 优点：攻击节点完整声明命中后效果，生成器和结算流程可以统一处理。
- 缺点：payload 扩大；伤害结算需要把结果路由回施法者侧。

## 决议

选 B：**固定模式的命中后效果作为攻击 payload 的一部分，由伤害结算流程统一触发。**

行为树继续描述技能宏观流程；只有确实需要根据命中结果继续分支的技能，才使用显式 proc 订阅。

## 代价

- 伤害结算从纯目标侧处理扩展为需要回流施法者侧事实。
- `onHitEffects` 与显式 proc 两种机制会并存，必须以“固定声明”与“复杂分支”作为选择边界。
- 未来若命中结果参与长流程控制，仍需显式事件或行为树分支。

## 影响范围

- 攻击契约需要容纳命中后效果声明。
- 伤害结算负责发布命中事实并触发声明式效果。
- 技能导入器可以直接生成 `onHitEffects`，不再为固定模式构造订阅流程。
