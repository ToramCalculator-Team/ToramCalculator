# 0033 - 只用被接纳的语义行动生成成员流程

- **状态**: Superseded by 0043
- **日期**: 2026-07-12
- **决策层**: 跨层（输入 / 引擎 / 运行分析 / 持久设计）
- **相关 ADR**: Superseded by 0043

## 决策问题

手动操作既包含被运行时接纳的行动，也包含因冷却、资源、状态或目标条件而被拒绝的尝试。系统需要决定运行分析与成员流程生成是否共用一份记录，还是分别保存输入轨迹和可生成流程的行动录制。

## 决策驱动

- 分析需要保留被拒绝的输入。
- 持久成员流程只能来自运行时实际接纳的语义行动。
- 运行产出写回持久设计必须由用户显式触发。

## 候选方案

### A. 录制原始输入并直接生成流程

- 优点：记录入口简单，保留全部操作尝试。
- 缺点：被拒绝输入会污染流程，输入发生也不能证明行动已被运行时接纳。

### B. 分别保存输入轨迹和行动录制

- 优点：分析保留全部输入，流程只消费被接纳行动。
- 缺点：同一次行动接纳需要维护两份可能漂移的事实。

## 决议

选择 B：**运行产出分别保留输入轨迹和行动录制，只有运行时权威边界接纳的语义行动才能进入行动录制。**

- 输入送达不等于行动接纳；接纳事实必须由实际决定行动开始或入队的运行时边界产生。
- 行动录制使用运行内逻辑时间，不使用界面事件时间作为权威时序。
- 行动录制属于会话内运行产出，只有用户显式保存时才能转换并写入成员流程。

## 代价

输入轨迹和行动录制重复表达行动接纳，需要额外的一致性协议。后续 ADR 0043 以单一最终输入事实取代了这项双轨保存决议。
